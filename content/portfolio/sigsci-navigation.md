## From startup nav to enterprise platform

Signal Sciences grew from a single-product security tool to an enterprise platform with dozens of features, multiple user roles, and complex multi-site configurations. The original navigation was unplanned and messy — built for early-stage growth, not enterprise scale.

Problems:
- Early stage, focused on growth — navigation was an afterthought
- Didn't reflect the information architecture
- Difficult to find key features
- No room to grow as the product expanded

![The original flat navigation](/images/portfolio/sigsci-navigation/old-nav.png)

## Map the product, find the users, follow the clicks

Before redesigning, the full site inventory was mapped and power users were identified across customer segments. Usage data was pulled to understand which features were actually accessed and how often.

Key research inputs:
- Full navigation inventory with feature descriptions — every item in the nav catalogued and defined
- Individual dashboard usage data (sessions per user, by number of sites managed) — revealed a power-user tier with dramatically higher engagement
- Click data on nav links showing where users actually went: Overview (23%), Agents (19%), Requests (14%) as the top three — everything else a long tail

This data drove prioritization: surface what's actually used, group what belongs together, bury what isn't core.

## A nav that reflects how the product is actually structured

The redesign restructured navigation around the product's real information architecture — corp level above, site level below — and grouped features by workflow rather than listing them flat.

Key changes:
- Breadcrumb navigation for wayfinding in hierarchy (Corp / Site name always visible)
- Surfaced frequently accessed features at the top level
- Grouped related features into menus by workflow (Monitor, Configure, Site Settings)
- Blocking status surfaced as a persistent inline indicator

![Navigation redesign — events page](/images/portfolio/sigsci-navigation/navigation-events.png)

## Built to grow

As Signal Sciences expanded from individual business units to full enterprise accounts, the navigation scaled with it. A second nav layer was introduced to reflect cross-enterprise management — corp-level tools and controls distinct from site-level operations.

- Refined navigation layer to reflect the growing enterprise tier
- New elements surfaced cross-enterprise features and management (Library, Corp Tools, My Profile at the global level)
- The two-layer model (global bar + contextual site nav) held up as the product grew

![Navigation after the redesign](/images/portfolio/sigsci-navigation/navigation-after.png)

## Startup to enterprise — without breaking the mental model

- Redesigned navigation for large, complex enterprise instances managing dozens of sites
- Scaled the platform from startup to enterprise without disrupting existing user workflows
- Contributed to platform readiness for Fastly acquisition ($825M)
